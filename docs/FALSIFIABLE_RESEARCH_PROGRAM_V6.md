# Falsifiable Research Program v6

Intentia must not be a myth machine. It must be a hypothesis machine.

## Claim boundary

Every claim gets one of four statuses:

```text
CAN_CLAIM
EVIDENCE_CANDIDATE
CONJECTURE
FORBIDDEN_WITHOUT_DATA
```

## Core hypotheses

Generated by `aris-falsification-report`.

1. Co-regulation: check-ins reduce unresolved conflict duration.
2. Reality fidelity: fact/inference/unknown labels reduce unsupported partner interpretations.
3. Consent integrity: consent-gated advice preserves autonomy while intimacy increases.
4. Memory continuity: event-sourced memory reduces rupture amnesia.
5. Value creation: investability appears only when reproducibility, consent and product metrics are measurable.

## Falsification standard

A system that cannot lose a claim is not scientific.

Intentia loses a claim when:

- boundary violations increase;
- unsupported claims do not decrease;
- repair latency does not improve;
- partner self-reports contradict system summaries;
- metrics cannot be reproduced from the evidence bundle.

## Minimum research stack

```text
baseline period
intervention period
manual coding guide
blind audit sample
reproducible script
confidence interval
negative result log
```

Negative results are assets. They make the repo credible.
