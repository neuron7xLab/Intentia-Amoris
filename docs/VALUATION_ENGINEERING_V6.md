# Intentia Amoris v6 — Valuation Engineering

## Position

A $23M claim is not impossible. It is just not a *replacement-cost* claim.
It must be converted into one of four defensible financial stories:

1. **Revenue asset** — ARR × multiple.
2. **Strategic IP asset** — acquisition value because the buyer cannot recreate the category quickly.
3. **Dataset moat** — consented, unique, longitudinal, multimodal data rights.
4. **Option asset** — right to build the first Relationship Intelligence OS before neurointerfaces make it obvious.

## Financial first principles

```text
price_of_love = undefined
value_of_repo = reproducible claims × rights × product surface × market optionality × trust
```

```text
if consent_integrity = 0
then archive_transferability = 0
then dataset_value = 0
```

```text
if no revenue and no LOIs
then 23M is strategic-option value
not fair-market value
```

## Valid valuation methods

- Cost approach: replacement cost of research, code, data engineering, governance and archive parsing.
- Income approach: discounted expected cash flows from productized Relationship Intelligence OS.
- Market approach: ARR/comparable multiples if the product has traction.
- Relief-from-royalty: avoided licensing cost for a proprietary relationship-intelligence protocol.
- Real options: staged value of future capabilities: telemetry, multimodal memory, neurointerface continuity, digital twin capsules.

These are implemented in `src/intentia_amoris/finance/valuation_model.py`.

The product is the **consented intelligence layer** that turns private human continuity into safe, useful, measurable action.
