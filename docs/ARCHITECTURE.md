# Intentia Amoris Architecture

## Principle

```text
The system is not an oracle.
The system is a living ledger of events, uncertainty, and value.
```

## Layers

```text
1 Input
  Telegram
  FastAPI
  future mobile app
  health telemetry
  media upload

2 Event Gateway
  validates privacy
  validates consent
  timestamps
  embeds

3 Memory
  immutable events

  media assets
  telemetry series
  profiles
  consent records

4 Plastic Kernel
  prototype similarity
  dynamic scales
  homeostasis
  value function

5 Agent Duet
  Aletheia = evidence and reality
  Eunoia = warmth and tenderness

6 Advice
  consent-gated
  uncertainty-aware
  RAG-informed
```

## Why vectors

Hard lexicons fail because each couple has private semantics.

In Intentia Amoris, “tenderness” is learned from:

- your remembered scenes;
- your own writing;
- Dasha’s own answers;
- photos and captions;
- repeated moments of safety;
- explicit golden states.

## Why event sourcing

A relationship is temporal.  
If the system only stores final summaries, it loses the living path.

Events are immutable.  
Beliefs are mutable.  
Scales are state.  
Advice is disposable.
